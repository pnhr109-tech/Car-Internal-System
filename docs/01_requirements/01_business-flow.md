# 車両買取業務フロー（裏議・ステータス管理）

## 全体フロー概要

```
アポイント
    ↓ （没 → 終了）
商談（査定）
    ↓ （没 or 査定前キャンセル → 終了）
契約
    ↓
車両引き取り
    ↓
出品準備
    ↓
陸送
    ↓
オークション ─→ 流れ・不備 ─→ WEB掲載
    ↓               ↓（再出品）↗
自社入金 ←──────────────────
    ↓
お客様へ支払い
    ↓
完了
```

**担当区分**
- 青（営業フェーズ）: アポイント / 商談 / 契約
- 緑（受注フェーズ）: 車両引き取り / 出品準備 / 陸送 / オークション / 自社入金 / お客様へ支払い

---

## 各ステップ詳細

### ステップ1: アポイント

| 項目 | 内容 |
|------|------|
| 担当 | 営業担当者 |
| 起動トリガー | 顧客からの査定申込（ナビクル / マイカースカウト / メール / HP依頼 / 来店 / 紹介 / カービュー） |

**必須入力・確認項目**
- 顧客情報（名前・電話番号）
- 車両情報（メーカー・車種・年式・走行距離）
- 査定希望日
- チャネル種別の確認

**承認**: 不要

**ステータス一覧**
- `予約` → `対応中` → `対応済`
- 離脱系: `接続不良` / `架電禁止` / `良` / `ユーザーキャンセル`

**次ステップへの進行条件**
- 顧客情報が登録済
- 車両情報が自動収集フラグON
- 査定日時が確定
- チャネルのステータスが「対応済」以上

**キャンセル・差し戻し**
- 顧客が査定を辞退 → ステータス「没」
- キャンセル理由を記録
- 架電禁止・不良は理由を記録

**備考**: ナビクル・マイカースカウトは外部サービスIDを紐付けて管理

---

### ステップ2: 商談（査定）

| 項目 | 内容 |
|------|------|
| 担当 | 営業担当者 |
| 起動トリガー | アポイント確定（予約の査定日時到来 + ステータス「対応済」） |

**必須入力・確認項目**
- 査定チェック項目（傷・修復歴 / 内装 / タイヤ）全項目確認
- 市場相場価格の入力
- 総合評価（1〜5）
- 車両追加属性（車台番号・修復歴・車検有効期限等）

**承認**: 要（マネージャー / 次席）— 承認者の承認が必要

**ステータス一覧**
- `査定中` → `成約` / `不成約` / `管理`
- `契約` / `没` / `再アプローチ`

**次ステップへの進行条件**
- 査定ステータスが「成約」
- 査定・市場相場価格が入力済
- 顧客が買取価格に口頭同意

**キャンセル・差し戻し**
- 顧客が価格に同意しない → ステータス「不成約」
- キャンセル理由記録 → 再アプローチ
- ステータス変更の際はマネージャーの重複確認が必要

---

### ステップ3: 契約

| 項目 | 内容 |
|------|------|
| 担当 | 営業担当者 |
| 起動トリガー | 査定ステータスが「成約」（顧客が買取価格に同意） |

**必須入力・確認項目**
- 買取契約書の作成・署名
- 本人確認書類（運転免許証等）の確認・ファイル登録
- 顧客口座情報の取得・登録
- 付帯費用リスト（支払承諾証明書等）の確認
- 支払い予定日（仮）の入力
- 出品予定金額の入力
- 追加費用の有無（追加内容コメント記録）
- 所有権解除の有無の入力（ある場合はパターンA/Bを選択し進捗管理）

**承認**: 要（社長 / マネージャー / 次席）

**ステータス一覧**
- 買取契約: `確認中` → `確認済` / `棄却`
- 未契約 → `契約済` / `棄却`

**次ステップへの進行条件**
- 買取契約書ステータスが「確認済」
- 本人確認書類が「確認済」以上
- 引き取り日時・場所の記録済
- 関連確認書類の「確認済」
- 所有権解除フラグが確認済（ある場合は解除管理ステータスが適切であること）

**キャンセル・差し戻し**
- 顧客が契約を破棄 → ステータス「破棄」、キャンセル日・理由を記録
- 再価格が必要な場合 → 訂正後に買取価格を再入力し再承認

**備考**: 後日品目（委任状等）は引き取り前受領可。受領完了は引き取り時に記録。

---

### ステップ4: 車両引き取り

| 項目 | 内容 |
|------|------|
| 担当 | サポート担当 |
| 起動トリガー | 買取契約ステータスが「契約済」 |

**必須入力・確認項目**
- 最終確認走行距離の記録
- 必須書類（後日送付ステータス）の受領確認
- 後日品目送付ステータスの更新
- 引き先確認書の印刷・各テスクへのフラグ（必要な場合は引き取り時に印刷）

**承認**: 不要

**ステータス一覧**: `未済` → `済`

**次ステップへの進行条件**
- 引き取りステータスが「済」
- 最終確認走行距離が入力済
- 後日品目送付ステータスが「確認済」以上
- 後日品目ストアIDが入力済であること

**キャンセル・差し戻し**
- 車両状態が査定時と著しく違う場合 → 差し戻し（②商談に戻り再査定）
- 入庫日変更が発生した場合も、変更後の入庫日を必ず記録

---

### ステップ5: 出品準備

| 項目 | 内容 |
|------|------|
| 担当 | 営業担当者 / サポート担当 |
| 起動トリガー | 引き取りステータスが「済」 |

**必須入力・確認項目**
- 出品準備の完了フラグON
- 車両情報（修復歴等）の確認/修正・整備記録
- 整備が必要な場合: 整備記録書が登録済
- 付帯費用（クリーニング/撮影等）の登録
- 出品表作成フラグON（出品表は外部サービス）
- 付帯費用のURL入力で作成完了、URLを本システムに手入力

**承認**: 要（サポートマネージャー）— 承認者の承認が必要

**ステータス一覧**: `準備中` → `完了`

**次ステップへの進行条件**
- 出品準備の完了フラグがON
- 整備内容が予算範囲内
- 整備が必要な場合は整備記録書が登録済かつ出品表URLが入力済であること

**キャンセル・差し戻し**
- 整備費用が予算超過 → 承認にて進行停止 → 承認後に再開
- 出品表作成は外部サービスを使用。管理・整備は外部サイト、内部システムはURLと有無フラグの管理のみ行う

---

### ステップ6: 陸送

| 項目 | 内容 |
|------|------|
| 担当 | サポート担当 |
| 起動トリガー | 出品準備完了フラグがON |

**必須入力・確認項目**
- 陸送業者の選択（業者IDの入力）
- 受地・到着地（オークション会場/店舗）の記録
- 陸送予定日の確定
- 陸送費用の記録
- 陸送前に陸送会社へ送付する車両リストの確認・チェック

**承認**: 不要

**ステータス一覧**: `後送` → `輸送中` → `完了`

**次ステップへの進行条件**
- 陸送ステータスが「完了」
- 陸送業者担当日が記録済

**キャンセル・差し戻し**
- 陸送中の事故・破損 → 差し戻し、対応方針を記録
- 陸送業者マスタで業者情報を事前に登録しておくこと
- 陸送依頼前にオークション会社のリストとシステムの内容を照合し、陸送会社と内容をすり合わせること

---

### ステップ7: オークション

| 項目 | 内容 |
|------|------|
| 担当 | （未定） |
| 起動トリガー | 陸送ステータスが「完了」 |

**必須入力・確認項目**
- 出品会場の選定（会場IDの入力）
- 考慮価格の設定
- 出品者の記録
- 出品結果（落札/流れ/不備）の記録
- オークション出品日の確定
- 流れ・不備の場合: 対応方針の選択

**承認**: 要

**ステータス一覧**
- 【出品】: `出品結果` → `落札` / `流れ` / `不備`
- 【結果】: `落/売` / `流れ` / `不備`

**次ステップへの進行条件**
- 出品結果が「落札」→ ①自社入金へ
- 流れの場合: 連絡流れの場合は顧客への案内が必要
- 落札手数料・成約手数料の記録済
- 再出品の場合: ⑤陸送に戻る（または会場で再出品）

---

### ステップ8: 自社入金

| 項目 | 内容 |
|------|------|
| 担当 | （未定） |
| 起動トリガー | 出品結果が「落札」またはWEB掲載ステータスが「成約」 |

**必須入力・確認項目**
- 入金元別の確認（オークション/WEB）
- 入金予定日・入金予定金額の記録
- 実入金日・実入金金額の記録・記録
- 入金額と入金予定金額の照合

**承認**: 不要

**ステータス一覧**: `入金記録` → `未確認` → `確認済`

**次ステップへの進行条件**
- 入金ステータスが「確認済」
- 実入金額が入金予定金額と一致（または誤差が許容範囲内）

**キャンセル・差し戻し**
- 入金額に差異がある場合 → 差異原因を調査・備考に記録
- 差異が大きい場合は経理担当者へエスカレーション

---

### ステップ9: お客様へ支払い

| 項目 | 内容 |
|------|------|
| 担当 | （未定） |
| 起動トリガー | 入金確認ステータスが「確認済」 |

**必須入力・確認項目**
- 顧客口座情報の確認（銀行行名/支店名/口座番号/口座種別）
- 支払い予定日の確定
- 支払い金額の照合（買取契約価格と異なる場合は上長の承認が必要）
- 振込実施・支払い日の記録

**承認**: 条件付き要（支払い金額が買取契約価格と異なる場合は社長/経理）

**ステータス一覧**: `支払い記録` → `支払い` → `支払済`

**次ステップへの進行条件**
- 支払い日が記録済

**キャンセル・差し戻し**
- 口座情報に誤りがある場合 → 差し戻し、顧客への確認
- 支払い後は顧客への完了通知を行う（取引・連絡履歴に記録）
- 再度支払い処理を実施

---

### ステップ10: Web掲載

| 項目 | 内容 |
|------|------|
| 担当 | （未定） |
| 起動トリガー | オークション経緯が「流れ・不備」かつ対応方針が「Web掲載移行」 |

**必須入力・確認項目**
- 掲載媒体の選定（媒体IDの入力）
- 掲載開始日・終了/打了の設定
- 車両画像・情報の最終確認

**承認**: 要（社長）— 掲載価格（特にオークション落札額より低い場合）は上長の確認・承認

**ステータス一覧**: `Web掲載` → `掲載中` → `成約` / `取り下げ`

**次ステップへの進行条件**
- 【成約の場合】: 成約 → ①自社入金へ（WEB媒体組入）
- 【取り下げの場合】: 対応方針を決定（再オークション/業者売却）

**備考**: 複数媒体への同時掲載を管理し、一つで成約したら他の媒体の掲載を取り下げること
