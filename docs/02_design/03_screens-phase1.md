# 画面一覧 — フェーズ1（アポイント〜契約）

## 画面構成図

```
サイドメニュー
├── ダッシュボード（既存・工事中）
├── 査定申込
│   ├── 査定申込一覧
│   ├── 査定申込手動入力（新規作成）
│   └── 査定申込詳細・変更
│           └── [商談に進む] → 査定登録
├── 案件一覧（商談昇格済みの申込）
│   └── 案件詳細（タブ構成）
│           ├── タブ: 査定（商談）
│           └── タブ: 契約
└── 契約一覧（契約済みのみ）
        └── 契約詳細（＝案件詳細の契約タブと共通）
```

---

## 画面一覧

### S01 査定申込一覧

| 項目 | 内容 |
|------|------|
| URL | `/leads/` |
| アクセス権限 | 全ロール（営業は自分のみ、管理者は全件） |
| 対応機能 | 1-1 申込一覧表示 |

**表示カラム:**
- 申込番号 / チャネル / 顧客名 / 電話番号 / 車両（メーカー・車種） / 査定予約日時 / 対応ステータス / 担当者 / 申込日時

**操作:**
- 絞り込み検索（チャネル / ステータス / 担当者 / 日付範囲 / 顧客名・電話番号）
- 行クリック → S02 申込詳細へ
- [新規作成] ボタン → S03 申込手動入力へ

---

### S02 査定申込詳細・変更

| 項目 | 内容 |
|------|------|
| URL | `/leads/<id>/` |
| アクセス権限 | 全ロール |
| 対応機能 | 1-3 申込詳細・変更、1-4 商談昇格、7-1 履歴追記 |

**表示・編集内容:**
- 申込情報（チャネル / 申込番号 / 査定予約日時）
- 顧客情報（氏名 / 電話番号 / メール）
- 車両情報（メーカー / 車種 / 年式 / 走行距離）
- 対応ステータス変更
- 担当者アサイン
- キャンセル理由入力（没処理時）
- 取引・連絡履歴の一覧 + 追記フォーム

**操作:**
- [情報を編集] → インライン編集 or モーダル
- [没にする] → キャンセル理由入力 → ステータス更新
- [商談に進む] → 顧客・車両の確認/紐付け → S04 案件詳細（査定タブ）へ

---

### S03 査定申込手動入力

| 項目 | 内容 |
|------|------|
| URL | `/leads/new/` |
| アクセス権限 | 全ロール |
| 対応機能 | 1-2 申込手動入力、2-1 顧客登録、3-1 車両登録 |

**入力フォーム:**
- チャネル種別（来店 / 紹介 / メール / HP / その他）
- 紹介者名（チャネル=紹介の場合のみ表示）
- 顧客情報：氏名 / 電話番号 / メール（既存顧客検索 or 新規入力）
- 車両情報：メーカー / 車種 / 年式 / 走行距離
- 査定予約日時
- 担当者
- 備考

---

### S04 案件詳細（商談・契約 一気通貫）

| 項目 | 内容 |
|------|------|
| URL | `/cases/<assessment_request_id>/` |
| アクセス権限 | 全ロール（営業は自分担当のみ） |
| 対応機能 | 4-1〜4-5、5-1〜5-6、6-1〜6-3、7-1・7-2 |

**ヘッダー部（常時表示）:**
- 顧客名 / 電話番号 / 車両（メーカー・車種・年式）
- 現在のフェーズ（査定中 / 成約 / 契約済 など）
- 担当者

**タブ構成:**

#### タブ①: 査定（商談）
- 査定日時・査定額・市場相場価格・総合評価
- 査定チェック項目（傷 / 修復歴 / 内装 / タイヤ）
- 車両追加情報（車台番号 / 修復歴 / 車検 / ミッション / ナンバー）
- ステータス表示・変更
- 承認状態（承認待ち / 承認済 / 差し戻し）
- [承認依頼] ボタン（担当者）/ [承認][差し戻し] ボタン（管理者）
- [契約へ進む] ボタン（成約後かつ承認済の場合のみ表示）

#### タブ②: 契約
- 契約日 / 買取確定価格（税抜・税込・消費税）
- 支払い予定日 / オークション出品予定日
- 加修フラグ・加修内容
- 所有権解除フラグ・解除進捗（パターンA/B・各ステータス）
- 金額訂正フラグ・訂正後価格
- 先払い入金記録
- 承認状態・承認操作
- ステータス表示（未契約 / 契約済 / 破棄）

#### タブ③: 書類
- 書類種別一覧（必須・任意）
- 各書類のステータス（未作成 / 受領待 / 受領済 / 確認済）
- 後日品送付ステータス
- 本人確認書類（アップロード・確認）

#### タブ④: 顧客・車両情報
- 顧客基本情報の確認・編集
- 口座情報の確認・追加・変更
- 車両基本情報の確認・編集
- 車両画像アップロード・一覧

#### タブ⑤: 連絡履歴
- 取引・連絡履歴の時系列一覧
- 新規履歴追記フォーム

|------|------|
| URL | `/approvals/` |
| アクセス権限 | マネージャー / 次席 / 社長 |

## 画面遷移フロー

```
S01 査定申込一覧
  ├─ [新規作成] → S03 申込手動入力 → S02 申込詳細
  └─ [行クリック] → S02 申込詳細
                        └─ [商談に進む] → S04 案件詳細（査定タブ）
                                              └─ [契約へ進む] → S04（契約タブ）

S05 案件一覧 → S04 案件詳細
S06 契約一覧 → S04 案件詳細（契約タブ）
S07 承認待ち一覧 → S04 案件詳細（該当タブ）
```

## URLルーティング設計（案）

```python
# leads/urls.py（既存を拡張）
urlpatterns = [
    path('leads/',              AssessmentRequestListView,   name='lead-list'),
    path('leads/new/',          AssessmentRequestCreateView, name='lead-create'),
    path('leads/<int:pk>/',     AssessmentRequestDetailView, name='lead-detail'),

    path('cases/',              CaseListView,                name='case-list'),
    path('cases/<int:pk>/',     CaseDetailView,              name='case-detail'),

    path('contracts/',          ContractListView,            name='contract-list'),

    path('approvals/',          ApprovalListView,            name='approval-list'),

    # API（ステータス更新・承認操作はPOSTで）
    path('api/leads/<int:pk>/promote/',  PromoteToCaseView,    name='api-promote'),
    path('api/cases/<int:pk>/approve/',  ApproveView,          name='api-approve'),
    path('api/cases/<int:pk>/reject/',   RejectView,           name='api-reject'),
]
```
